/// EDF tick — O(log 16), WCET ≈ 500 ns on Cortex-A53.
pub fn tick(&mut self, now: u32) -> Option<TaskId> {
    self.now_us = now;
    let task = self.heap.peek()?;

    if now > task.deadline_us {       // deadline miss
        let mut t = self.heap.pop().unwrap();
        t.miss_count += 1;
        Watchdog::signal_miss(t.id, t.miss_count);
        return None;
    }

    let task = self.heap.pop().unwrap();
    (task.entry)();                   // dispatch fn ptr
    Some(task.id)
}

FAQ

Common Questions

AxonOS is currently a research and engineering project — a working codebase, a rigorous technical specification, and a 33-article publication series that documents the design decisions in full. It is not yet a commercial product. The goal is to establish the correct architecture before optimising for deployment. The axonos-core codebase is public and buildable on Cortex-A53 and Cortex-M4F targets today.
Rust's ownership model eliminates the class of memory safety bugs (use-after-free, buffer overflows, data races) that are responsible for the majority of critical security vulnerabilities in C/C++ systems software. For a BCI kernel that processes data of the highest possible sensitivity, this is not a performance trade-off — it is a safety requirement. The #![no_std] constraint means we use no standard library, no heap allocator by default, and no OS abstractions — exactly what a bare-metal real-time kernel requires.
It means the privacy guarantee is enforced by the physical and type-theoretic structure of the system, not by policy or encryption alone. The EphemeralNeuralBuffer type has no save(), clone(), or transmit_raw() methods — they do not exist in the type system. The raw neural frame lives for ≤6 ms in DSP SRAM before being overwritten by the next DMA cycle. No software bug, no privilege escalation, and no policy violation can transmit data that was never stored.
The reference design uses an STM32F407 for the DSP pipeline (Cortex-M4F with hardware FPU) paired with a Cortex-A53 application processor for the kernel and application layer. The complete BOM including EEG amplifier, electrode interface, and ATECC608B cryptographic coprocessor is detailed in Article 31. Total component cost is approximately $55 in small-volume production. The firmware builds for thumbv7em-none-eabihf (Cortex-M4F) and aarch64-unknown-none (Cortex-A53) with a single cargo build command.
The ZeroCalib protocol (Article 32) uses three stages. First, a pre-trained universal classifier built from N=127 subjects using Riemannian MDM on Sym⁺(8). Second, Euclidean Alignment: 56 unlabelled resting-state covariance matrices (7.5 seconds) are collected to estimate the new user's manifold offset, then all covariances are normalised to the identity. This takes the theoretical 60-second warmup window and achieves 78.3% two-class accuracy. Third, online Riemannian gradient descent updates the class prototypes using high-confidence pseudo-labels from natural BCI use, reaching 91.7% within 8 minutes without any structured calibration task.
Yes — the axonos-core workspace is published under the MIT licence at github.com/AxonOS-org/axonos-core. The codebase includes all five crates (axonos-kernel, axonos-signal, axonos-intent, axonos-ipc, axonos-privacy), builds for both embedded targets, and includes cargo test for host-side unit tests.
